0B — Filtri di Testo
ComandoFlags / UsoCosa fa
grep -c conta le righe con match (stampa un numero, non le righe) · -n prefissa ogni riga con il suo numero di riga · -v inverte — restituisce righe che non corrispondono · -i case-insensitive · -o stampa solo la parte corrispondente, una per riga — conta occorrenze, non righe Filtra righe che corrispondono al pattern. Attenzione: senza -o conta righe; con -o conta occorrenze
Pattern regexSignificatoEsempio
. Qualsiasi carattere (uno) grep '.' → qualsiasi riga non vuota
\. Punto letterale (\ toglie il significato speciale) grep '\.' → righe con un punto nel nome
^ Ancora inizio riga grep '^root' → righe che iniziano con "root"
$ Ancora fine riga grep ':$' → righe che finiscono con ":"
^$ Inizio subito seguito da fine = riga vuota grep -v '^$' → elimina righe vuote
sort -n numerico · -r inverso · -rn num. inverso Ordina righe. Senza flag: alfabetico. Necessario prima di uniq perché uniq conta solo righe consecutive uguali
uniq -c mostra conteggio Rimuove righe duplicate consecutive (usare dopo sort)
wc -l righe · -w parole · -c byte Conta. wc -l < file = numero righe
rev Rovescia ogni riga carattere per carattere. Trucco: rev | cut -d'.' -f1 | rev estrae la parte dopo l'ultimo punto (utile per estensioni)
cut -d: -f1 Estrae campo N con delimitatore. Es: cut -d: -f1 /etc/passwd · cut -d'.' -f1 prende il primo campo prima del punto
tr -c complemento (opera su ciò che NON è in SET1) · -s squeeze (comprime sequenze consecutive del carattere sostituito in uno solo) · -d elimina i caratteri in SET1 senza sostituirli · classi POSIX: [:alpha:] lettere · [:upper:] maiuscole · [:lower:] minuscole Sostituisce caratteri: tr 'SET1' 'SET2'. Legge da stdin — usare con < o pipe, non accetta file direttamente.
Esempi:
tr '[:upper:]' '[:lower:]' < file — lowercase tutto · tr -cs '[:alpha:]' '\n' < file — una parola per riga (elimina punteggiatura e spazi) · tr -cd 'a' < file | wc -c — conta occorrenze di 'a'
head -N file Stampa prime N righe
tail -N file · -f file Ultime N righe. -f = segue in tempo reale (log!)
Pipeline classica: cat /etc/passwd | cut -d: -f1 | sort | uniq | wc -l ⚔ SEC — stessa logica per analizzare output Nmap/Suricata
BLOCK 1 Bash Scripting — Variabili · Condizioni · Loop · Funzioni · Case ▾
1A — Creare ed Eseguire uno Script
Comando / SintassiFlags / UsoCosa fa
#!/bin/bash prima riga obbligatoria Shebang — indica al SO di usare Bash come interprete
cat > script.sh << 'EOF' heredoc Crea il file da terminale. Tutto fino alla riga EOF finisce nel file
chmod +x script.sh oppure chmod 755 Rende lo script eseguibile
./script.sh Esegue lo script nella directory corrente
$(comando) command substitution Esegue e inserisce l'output inline. Es: echo "oggi: $(date)"
$((espressione)) aritmetica intera Calcola espressioni. Es: N=$((N + 1))

SintassiEsempioCosa fa

${#VAR} echo ${#NOME} Lunghezza della stringa

$# Numero totale di argomenti passati

$HOME $USER $SHELL $PATH Variabili d'ambiente — predefinite dal sistema
1A — Condizioni
SintassiOperatoriNote

Numeri -eq -ne -gt -lt -ge -le equal · not equal · greater · less · ≥ · ≤

File -f -d -e -r -w -x file regolare · directory · esiste · leggibile · scrivibile · eseguibile
[ ! cond ] Negazione — spazio tra ! e operatore obbligatorio
exit 0 / exit 1 Termina script con successo / errore. Usato in guard clause
1A — Loop
SintassiEsempioQuando usarlo

1B — Funzioni · Case · [[ ]] · Exit code
SintassiUsoNote
nome() { … } dichiarazione funzione Deve precedere la prima chiamata. Chiamata: nome arg1 arg2
local VAR=valore dentro la funzione Variabile visibile solo nella funzione — evita di sovrascrivere globali
return N dentro la funzione Exit code della funzione (0=ok, altro=errore). NON chiude lo script
$(funzione arg) cattura output Per restituire una stringa da funzione: echo val + command substitution
case $VAR in p) ;; esac switch su pattern ;; chiude branch · p1|p2) OR · *) default
[ c1 ] && [ c2 ] AND (parentesi singole) && e || stanno FUORI dalle parentesi
[[ c1 && c2 ]] AND (doppie parentesi) Doppie parentesi: && || dentro. Pattern: [[ "$VAR" == *pat* ]]
$? dopo ogni comando Exit code dell'ultimo comando. 0=ok, altro=errore. Si sovrascrive subito
if systemctl is-active --quiet ssh exit code diretto if valuta l'exit code di qualsiasi comando, non solo [ ]

2A/2B — Permessi Avanzati e Bit Speciali
Comando / BitNotazioneCosa fa
chown sudo chown utente:gruppo file Cambia owner e gruppo di un file
chgrp sudo chgrp gruppo file Cambia solo il gruppo
umask umask 0002 Permessi tolti ai nuovi file. 0002 → file 664 · dir 775 (gruppo scrive)
SUID 4xxx chmod 4755 · u+s Processo gira con UID del proprietario. Es: passwd → root ⚔ SEC
SGID 2xxx chmod 2770 · g+s Su dir: file creati dentro ereditano il gruppo della dir (collaborazione)
Sticky 1xxx chmod 1777 · +t Su dir world-writable: solo il proprietario può cancellare i propri file. Es: /tmp
Nella stringa permessi: SUID → rws al posto di rwx per owner · SGID → s per group · Sticky → t alla fine
⚔ SEC — find / -perm -4000 enumera tutti i SUID del sistema = potenziali vettori di escalation

2C — find — Ricerca nel Filesystem
Criterio / AzioneEsempioCosa fa

-name -name '*.log' Nome del file (wildcard tra apici)
-type -type f / -type d f=file · d=directory · l=link simbolico

2C — tar — Archiviazione
CombinazioneComandoCosa fa
Crea archivio tar cvpf archivio.tar /path c=crea · v=verboso · p=preserva permessi · f=file (deve essere ultima)
Estrai tar xf archivio.tar x=extract. Rileva compressione automaticamente
Estrai in dir tar -C /newdisk -xvpf archivio.tar -C cambia directory prima di estrarre
Crea + gzip tar czf archivio.tar.gz /path z=gzip · j=bzip2 · J=xz
Lista contenuto tar tvzf archivio.tar.gz t=table of contents — mostra senza estrarre
Pipeline senza file intermedio tar cvpf - /home/* | tar -C /newdisk -xvpf - - al posto del nome = stdin/stdout. Utile quando manca spazio

3A — systemctl — Controllo Servizi (runtime)
ComandoFlags / UsoCosa fa
systemctl start nomeservizio Avvia il servizio ora (non persistente)
systemctl stop Ferma il servizio
systemctl restart Stop + start
systemctl reload Ricarica configurazione senza fermare il processo

3A — systemctl — Configurazione Persistente & Ispezione
ComandoFlags / UsoCosa fa
systemctl enable nomeservizio Avvio automatico al boot (effetto al prossimo reboot)
systemctl disable Disabilita avvio automatico (ma avvio manuale rimane possibile)
systemctl mask Neutralizza completamente — impossibile avviare anche manualmente
systemctl list-units -t service Lista servizi attivi. Senza filtro: tutte le unit
systemctl list-unit-files -t service --state=enabled Lista servizi installati (anche inattivi) — utile per inventario
systemctl --state=failed Mostra unit in stato failed — diagnostica crash
systemctl get-default Target di avvio corrente (es. multi-user.target)
systemctl isolate multi-user.target Passa a quel target subito senza reboot
⚔ SEC — systemctl list-units -t service elenca la superficie d'attacco: stessi servizi che Nmap trova sulle porte aperte
